# Agent & Developer Context

This file provides context, detailed guidelines, and ready-to-use prompts for AI agents (and humans) working on `upup-web`.

## 1. Project Overview
- **Name**: upup-web
- **Framework**: Next.js 16.1 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Package Manager**: NPM
- **Main Directory**: `src/`

## 2. Tech Stack & Decisions
- **UI Framework**: React 19 (Functional Components only).
- **Icons**: `lucide-react` (Recommended).
- **State Management**: React Context / Hooks (Zustand if complex).
- **Video/Image**: standard `<img>` or `next/image` depending on optimization needs.
- **Validation**: `zod` for schema validation.
- **Fetching**: Server Components for data fetching where possible. `swr` or `react-query` for client-side if needed.

## 3. Coding Guidelines
- **Directory Structure**:
    - `src/app`: App Router pages and layouts.
    - `src/components`: Reusable UI components.
    - `src/lib`: Utility functions, helpers.
    - `src/hooks`: Custom React hooks.
- **Naming Conventions**:
    - Components: PascalCase (e.g., `UserProfile.tsx`).
    - Utilities: camelCase (e.g., `formatDate.ts`).
    - Folders: kebab-case (e.g., `user-profile`).
- **Best Practices**:
    - **Layout**: Always assume strict mobile width (max 500px). The root layout handles the constraint, but design components accordingly.
    - Use `const` over `let`.
    - Prefer **Server Components** by default. Add `'use client'` only when interactivity is needed (hooks, event listeners).
    - Implement "Early Returns" to reduce nesting.
    - Explicitly type props using interfaces/types.
    - Co-locate styles if not using standard Tailwind classes (rarely needed).
